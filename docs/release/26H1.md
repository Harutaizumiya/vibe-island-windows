# 26H1 版本简述

## 版本定位

`26H1` 是当前这一轮功能集成后的阶段版本。它已经把项目从“基础状态展示原型”推进到了“可持续跟踪 Codex CLI 当前会话，并具备基本调试能力”的状态。


## 当前已经实现的功能

### 1. 真实 Codex 会话跟踪

- 支持读取 `~/.codex/session_index.jsonl`
- 支持读取 `~/.codex/sessions/**/rollout-*.jsonl`
- 支持 watcher、活跃轮询和周期性补扫，降低新 session 漏发现和更新延迟问题
- 支持多 session 聚合，优先展示当前更活跃、更相关的会话

### 2. 状态机与状态展示

当前版本已经实现：

- `Idle`
- `Processing`
- `RunningTool`
- `Finishing`
- `Completed`
- `Stalled`
- `Interrupted`
- `Unknown`

并在内部引入了更细的推断能力：

- `ThinkingSuspected`
- `RunningToolLong`

这些推断态目前主要用于排序、超时和文案，不额外增加 UI 枚举复杂度。

### 3. 展开态内容分流

展开岛体时，内容会按状态切换：

- `Processing` / `Finishing`：展示 Agent 输出
- `RunningTool`：展示工具调用详情和命令
- `Completed`：展示已更改文件

这让展开区从“单一提示面板”升级成了一个与当前执行阶段相匹配的信息面板。

### 4. 交互行为与布局

- 支持悬浮自动展开
- 支持点击展开
- 支持失焦自动折叠，并区分悬浮展开和手动展开两套时间
- 支持展开区内容更新时重新计算高度
- 支持宿主窗口跟随展开内容同步增高，减少底部被裁切的问题
- 收缩态右侧加入中文状态显示
- `Idle` 状态使用 Codex 图标
- `Completed` 状态使用 `✅`

### 5. 调试与配置

- 支持 `mock` / `codex` 两种数据源
- 支持调试模式
- 调试模式下可展示当前状态与数据来源
- 配置从环境变量扩展到 `islandsettings.json`
- 配置文件已支持中文注释，方便直接手改

### 6. 日志与排障工具

- 主程序持续写入 `startup.log`
- 提供独立的 `DynamicIsland.Logger`
- 可分别记录 session 读取细节与岛体最终状态，便于排查延迟、漏更新和状态误判

## 当前版本的价值

这一版已经能满足以下使用场景：

- 在桌面上持续观察 Codex CLI 当前回合是否还在工作
- 快速区分“正在思考”“正在调工具”“已经完成”“可能卡住”“已中断”
- 在任务完成后直接看到已更改文件
- 在状态异常时，通过日志和调试模式判断问题出在数据源、状态机还是 UI

换句话说，当前版本已经具备“日常可用 + 可调试”的基础形态。

## 下一阶段计划

后续更新方向建议集中在这几类：

### 1. 数据准确性继续增强

- 进一步细化 rollout 事件分类，减少“有进展但 UI 看起来没动”的情况
- 补强对更多工具类型和输出类型的识别
- 继续压缩真实数据源下的首帧延迟和增量更新延迟

### 2. 展开区信息组织优化

- 优化长文本、多行命令和多文件列表的排版
- 给工具调用详情增加更清晰的结构化展示
- 在完成态中提供更有层次的变更摘要，而不只是文件名列表

### 3. 配置能力继续外置

- 把更多交互和视觉参数移入配置文件
- 为不同使用习惯提供更轻量的个性化调整空间

### 4. 发布与工程化完善

- 完善正式发布流程
- 整理发布产物结构
- 进一步补充文档与测试覆盖
- 视需要切换到更标准的测试框架

## 当前结论

`26H1` 可以看作这个项目的第一个“可长期带着跑”的阶段版本。它已经解决了最关键的几件事：能跟住 Codex 会话、能把状态稳定映射到岛体、能在出问题时留下足够多的线索。

后面的重点不再是“从 0 到 1 能不能跑起来”，而是“让状态更准、信息更清晰、发布更稳”。
